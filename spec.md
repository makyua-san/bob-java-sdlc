# 機能仕様: 備品購入申請システム（MVP）

## 1. 概要

### 1.1 機能名
備品購入申請・承認機能

### 1.2 目的（WHY）
社内の備品購入プロセスを効率化し、申請から承認までの流れを可視化・追跡可能にする。紙ベースやメールベースの申請では、承認状況の把握が困難で、却下理由の伝達も不明確になりがちである。本システムにより、申請者と承認者の双方が状況を把握しやすくなり、業務の透明性と効率性が向上する。

### 1.3 対象ユーザー
- **申請者**: 備品購入が必要な社員
- **承認者**: 購入申請を承認・却下する権限を持つ管理者

## 2. 機能要件（WHAT）

### 2.1 備品購入申請の登録

#### 2.1.1 申請内容
申請者は以下の情報を入力して備品購入申請を登録できる：

- **申請者情報**: 氏名、部署、連絡先（システムログインユーザーから自動取得）
- **備品情報**:
  - 備品名（必須、最大100文字）
  - 数量（必須、1以上の整数）
  - 単価（必須、0円以上）
  - 合計金額（自動計算: 数量 × 単価）
  - 購入理由（必須、最大500文字）
  - 希望納期（任意）
  - 備考（任意、最大500文字）

#### 2.1.2 申請の保存
- 申請は「下書き」状態で一時保存可能
- 「申請する」ボタンで正式に申請（ステータスが「申請済み」に変更）
- 申請後は申請内容の編集不可

#### 2.1.3 高額申請の識別
- **高額申請の定義**: 合計金額が設定された閾値（デフォルト: 50,000円）を超える申請
- 高額申請には以下の視覚的な注意表示を行う：
  - 警告アイコン（⚠️）を表示
  - オレンジ色（#FF9800）の背景色を適用
  - アクセシビリティ対応として、色だけでなくアイコンも併用することで色覚異常のユーザーにも識別可能にする
- 閾値は設定ファイルで管理し、ハードコーディングしない

### 2.2 申請の承認・却下

#### 2.2.1 承認操作
承認者は申請を確認し、以下のいずれかの操作を実行できる：

- **承認**: 申請を承認し、ステータスを「承認済み」に変更
- **却下**: 申請を却下し、ステータスを「却下」に変更

#### 2.2.2 却下理由の入力
- 却下時は**却下理由の入力が必須**
- 却下理由は最大500文字
- 空文字またはnullの場合はエラーとし、却下操作を受け付けない

#### 2.2.3 権限制御
- 承認・却下操作は**承認権限を持つユーザーのみ**実行可能
- 承認権限のないユーザーがアクセスした場合はエラーメッセージを表示

### 2.3 申請一覧の確認

#### 2.3.1 申請者向け一覧
申請者は自分が登録した申請の一覧を確認できる：

- 表示項目: 申請日、備品名、合計金額、ステータス、承認者名（承認・却下後）
- ステータスによる絞り込み: 下書き / 申請済み / 承認済み / 却下
- 却下された申請は却下理由を確認可能

#### 2.3.2 承認者向け一覧
承認者は承認待ちの申請一覧を確認できる：

- 表示項目: 申請日、申請者名、備品名、合計金額、ステータス
- ステータスによる絞り込み: 申請済み / 承認済み / 却下
- 高額申請には注意表示

### 2.4 ステータス遷移

申請のステータスは以下の順序でのみ遷移する（逆方向遷移は不可）：

```
下書き → 申請済み → 承認済み
                  ↘ 却下
```

- **下書き**: 申請者が作成中、まだ正式に申請していない状態
- **申請済み**: 正式に申請され、承認者の確認待ち
- **承認済み**: 承認者が承認した状態
- **却下**: 承認者が却下した状態

## 3. 非機能要件

### 3.1 パフォーマンス
- 申請一覧の表示: 1秒以内
- 申請登録・承認・却下操作: 2秒以内

### 3.2 セキュリティ
- 認証: ユーザーはログイン必須
- 認可: 承認操作は承認権限を持つユーザーのみ
- 入力検証: SQLインジェクション、XSS対策を実施

### 3.3 ユーザビリティ
- 高額申請は視覚的に識別しやすくする
- 却下理由は申請者に明確に伝わるよう表示
- エラーメッセージは具体的で理解しやすい内容

### 3.4 保守性
- 高額申請の閾値は設定ファイルで管理
- ログは構造化形式（JSON）で出力し、追跡可能にする

## 4. 制約事項

### 4.1 技術制約
- Java 21 / Spring Boot 3系 / Maven
- RESTful API + サーバーサイドレンダリング
- データベース: リレーショナルDB（PostgreSQL想定）

### 4.2 業務制約
- MVP範囲のため、以下は対象外：
  - 複数承認者による段階的承認
  - 申請の取り下げ・再申請
  - 承認後の発注管理
  - 予算管理機能

## 5. 受け入れ条件

### 5.1 申請登録
- [ ] 申請者が必須項目を入力して申請を登録できる
- [ ] 合計金額が自動計算される
- [ ] 下書き保存ができる
- [ ] 申請後は編集できない

### 5.2 高額申請
- [ ] 閾値を超える申請に注意表示が出る
- [ ] 閾値は設定ファイルで管理されている
- [ ] 閾値の変更が即座に反映される

### 5.3 承認・却下
- [ ] 承認者が申請を承認できる
- [ ] 承認者が却下理由を入力して却下できる
- [ ] 却下理由が空の場合はエラーになる
- [ ] 承認権限のないユーザーは操作できない

### 5.4 申請一覧
- [ ] 申請者が自分の申請一覧を確認できる
- [ ] 承認者が承認待ち申請一覧を確認できる

- [ ] 却下理由が確認できる

### 5.5 ステータス遷移
- [ ] ステータスが正しい順序でのみ遷移する
- [ ] 逆方向遷移はエラーになる

## 6. 画面イメージ（概要）

### 6.1 申請登録画面
- 入力フォーム（備品名、数量、単価、購入理由等）
- 合計金額の自動表示
- 高額申請の場合は警告表示
- 「下書き保存」「申請する」ボタン

### 6.2 申請一覧画面
- 申請リスト（テーブル形式）
- ステータス絞り込み
- 高額申請の視覚的識別

### 6.3 承認・却下画面
- 申請詳細表示
- 「承認」「却下」ボタン
- 却下理由入力欄（却下時のみ表示）
